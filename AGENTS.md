# Project Agent Rules

## Model routing

- 所有代码修改、测试、代码审查、重构、调试和验证任务，自动调用 `$codex-auto-model-router` 的 Apply 路径。
- Apply 路径为整次用户请求只选择一个模型和推理强度，不自动拆成多模型任务段，也不先插入 Assess 回合。
- 报告覆盖当前请求时复用对应路由；报告缺失、过期或未覆盖时，使用 Skill 定义的确定性 Apply 默认路由，不阻塞实际工作。
- 速度优先：当前模型和推理强度已经匹配时直接执行；需要切换时只续接一次，并在可靠元数据可用时于完成后恢复原模型和推理强度。
- 单文件机械小改如果切换与恢复成本高于任务本身，保持当前模型直接完成；用户明确指定模型时不应用此优化。
- 仅使用 `CODEX_THREAD_ID` 和对应 session 元数据识别当前任务及原模型；原模型或推理强度不可验证时，不执行会污染后续回合的同任务持久切换。
- Apply 开始前在当前 Codex 对话框显示一次模型和推理强度；命令、文件、验证和恢复阶段不重复提示。优先使用 Codex 原生同任务模型覆盖，不创建新的顶层任务。
- 如果所选模型不可用，依次尝试显式可选模型的子智能体和当前模型，并按 `codex-auto-model-router` 的回退规则记录实际可验证的模型；不得把配置模型冒充成实际运行模型。
- 简单问答、文案确认、解释说明和只读查询不走 Apply 路径，也不启动模型路由，以避免额外延迟。
- 只读查询模型使用比例或记录已完成任务时，使用 `codex-auto-model-router` 的 Query/Record 快速路径。
- 默认只在本地修改和验证，不自动执行 `git commit` 或 `git push`；只有用户明确要求提交、推送或发布到 GitHub 时才执行。

## Verification

- 遵守本仓库已有的验证说明；验证范围与任务风险相称。
- 每个 Apply 任务完成后，报告修改文件、执行的检查和剩余风险；项目本地路由账本不可写时不阻塞主任务。
