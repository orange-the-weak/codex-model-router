# Security Policy

## Reporting a vulnerability

Do not post credentials, private prompts, source code, or exploit details in a public issue. Use GitHub's private vulnerability reporting or security-advisory feature for the repository.

Include the affected file, impact, reproduction conditions, and a minimal proof of concept that contains no third-party secrets or data.

## Scope

Security reports may cover installer behavior, unsafe filesystem writes, ledger data exposure, command injection, report corruption, or routing behavior that falsely claims a model or verification result.
